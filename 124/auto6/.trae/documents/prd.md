# 恋爱养成游戏产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
创建一个单文件纯原生HTML恋爱养成游戏，包含完整的对话树系统、角色属性管理、多分支剧情和存档功能。

### 1.2 目标用户
- 喜欢文字冒险类游戏的玩家
- 希望体验轻量级恋爱养成游戏的用户
- 对移动端友好的网页游戏感兴趣的用户

### 1.3 核心价值
- 提供沉浸式的恋爱养成体验
- 通过选择影响剧情走向和角色关系
- 支持进度保存和恢复

## 2. 功能需求

### 2.1 对话树系统
- 多分支选项设计
- 选择即时影响好感度变化
- 好感度数值在界面可见
- 不同好感度触发不同对话内容

### 2.2 角色系统
- 可攻略角色：姓名、好感度(0-100)、心情值(-10-10)
- 可视化角色状态面板
- 进度条+数值实时展示
- 数据变化过渡动画

### 2.3 模块化架构
- 角色数据模块
- 对话逻辑模块
- UI渲染模块
- 存档管理模块
- 各模块通过明确接口协同

### 2.4 结局系统
- 好感度≥80：恋人结局
- 好感度≤20：陌路结局
- 全屏弹窗展示结局
- 重置游戏选项
- localStorage存档
- 数据完整性检测

### 2.5 响应式UI
- flex/grid布局
- 角色立绘区域
- 对话框（角色名+对白）
- 选项按钮区
- 状态栏
- 移动端适配

### 2.6 事件触发系统
- 好感度首次进入特定区间触发特殊邀约
- 邀约对话UI与普通对话有明显区别

### 2.7 心情值系统
- 心情值影响选项可用性
- 心情较低时隐藏/禁用亲密选项

### 2.8 防快速点击机制
- 节流或按钮禁用机制
- 按钮置灰视觉反馈

### 2.9 日常问候机制
- 随机早安/晚安问候语
- 内容受好感度和心情值影响
- 至少5种问候变体

## 3. 技术约束
- 纯原生HTML+CSS+JavaScript
- 无第三方库或框架
- 单文件内联所有代码
- 兼容Chrome和Edge最新版
- 无外部资源引用

## 4. 设计风格
- 浪漫温馨的配色方案
- 柔和的过渡动画
- 清晰的视觉层次
- 移动端优先的响应式设计
