## 1. 产品概述
即时战略游戏单文件HTML应用，玩家通过采集资源、建造建筑、生产单位与AI敌人进行对战。
- 核心玩法：资源采集→建筑建造→单位生产→战术进攻
- 目标用户：喜欢经典RTS游戏的玩家，无需安装即可游玩

## 2. 核心功能

### 2.1 功能模块
1. **游戏主界面**：Canvas战场渲染、HUD信息显示、小地图
2. **单位控制系统**：选择、移动、攻击、编队
3. **资源系统**：金币、木材采集与管理
4. **建造系统**：建筑放置与建造
5. **AI系统**：敌方自主采集、生产、进攻

### 2.2 功能详情
| 功能模块 | 子功能 | 功能描述 |
|---------|--------|---------|
| 战场渲染 | 网格地图 | 80×60网格，可通行/不可通行地形标记 |
| | 单位渲染 | 工人、战士、建筑的绘制与状态显示 |
| | 视觉效果 | 选择框、血条、进度条等 |
| 输入控制 | 鼠标操作 | 左键选择/框选、右键移动/攻击 |
| | 键盘操作 | A攻击移动、S停止、编队控制 |
| 单位系统 | 工人 | 采集资源、建造建筑 |
| | 战士 | 战斗、攻击移动 |
| 资源系统 | 采集 | 金矿、树木资源采集 |
| | 存储 | 资源返回基地入账 |
| 建造系统 | 建筑放置 | 可建造位置校验 |
| | 建造进度 | 可视化进度条 |
| AI系统 | 资源采集 | AI工人自主采集 |
| | 单位生产 | 自主生产工人和战士 |
| | 进攻策略 | 组织部队进攻玩家基地 |
| 小地图 | 全局预览 | 地形、单位、建筑的简略显示 |
| | 视角跳转 | 点击小地图移动主视角 |

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深绿/深灰军事风格，金色/红色强调色
- 界面风格：复古RTS风格，像素感线条
- 字体：等宽字体，清晰易读
- 布局：Canvas占主要区域，HUD在顶部和右侧

### 4.2 界面布局
| 区域 | 位置 | UI元素 |
|-----|------|--------|
| 资源HUD | 顶部 | 金币、木材、人口显示 |
| 主战场 | 中心 | Canvas游戏画面 |
| 小地图 | 右下角 | 全局地图预览 |
| 选中信息 | 左下角 | 选中单位/建筑详情 |
| 建造菜单 | 左侧 | 建筑建造选项 |

### 4.3 性能要求
- 分辨率：1280×720
- 同屏单位：≥100个
- 帧率：稳定≥30FPS
