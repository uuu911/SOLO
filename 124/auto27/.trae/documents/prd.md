# 2D沙盒开放世界游戏原型 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
创建一个纯原生JavaScript + HTML5 Canvas的2D沙盒开放世界游戏原型，单HTML文件即可运行，无需任何依赖。

### 1.2 核心目标
- 提供一个可直接在浏览器运行的沙盒游戏体验
- 实现挖掘、建造、探索等核心沙盒玩法
- 保证良好的性能和用户体验
- 代码结构清晰，模块化设计

### 1.3 目标用户
- 游戏开发者学习参考
- 休闲玩家体验沙盒游戏乐趣

## 2. 功能需求

### 2.1 世界生成
- 使用噪声算法生成200x200网格的开放世界
- 每个网格32x32像素
- 地形类型：草地、泥土、石头、水域
- 玩家出生点位于平坦陆地中心

### 2.2 挖掘与建造系统
- 鼠标左键破坏方块，自动采集资源
- 鼠标右键放置当前选中方块
- 快捷栏UI支持4种方块：泥土、石头、木材、树叶
- 选中项高亮显示

### 2.3 移动与物理
- WASD控制移动
- 空格键跳跃
- 重力系统：玩家下落直至接触方块
- 碰撞检测：无法穿过方块
- 摄像机平滑跟随玩家

### 2.4 昼夜循环
- 白天30秒，夜晚15秒
- 夜晚叠加半透明暗色层
- 夜晚视野范围缩减至白天的60%

### 2.5 音效系统
- Web Audio API生成合成音效
- 挖掘和放置方块不同音效
- 循环环境背景音
- 声音开关控制

### 2.6 粒子效果
- 挖掘方块时生成粒子效果
- 自然的运动轨迹和生命周期

### 2.7 生物陪伴
- 友善生物（小猫）跟随玩家
- 自动绕开障碍物
- 增加游戏生气

### 2.8 窗口适配
- Canvas自适应窗口大小
- 画面不拉伸变形
- 失焦时暂停或降频运行

## 3. 技术需求

### 3.1 技术栈
- 纯原生JavaScript
- HTML5 Canvas
- Web Audio API
- 无第三方依赖

### 3.2 性能指标
- 稳定60FPS（requestAnimationFrame驱动）
- 内存占用<100MB
- 只渲染视口内方块

### 3.3 代码架构
- 模块化设计：World、Player、Renderer、AudioManager、InputHandler
- 模块间通过明确接口通信
- 禁止直接操作内部数据结构

## 4. 设计风格
- 像素风格，简约像素风
- 明亮清新的配色
- 清晰的视觉反馈

## 5. 交付标准
- 单个HTML文件，双击即可运行
- 所有功能完整实现
- 代码结构清晰，注释完整
- 性能达标
