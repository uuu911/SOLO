# 历史帝国决策游戏 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
创建一个回合制历史帝国决策游戏，玩家扮演古文明统治者，通过处理历史事件来管理国家资源，体验文明发展的历程。

### 1.2 项目目标
- 实现一个完整可运行的单页HTML游戏
- 提供沉浸式的历史决策体验
- 展示真实历史事件改编的游戏内容
- 实现事件连锁记忆效应机制

### 1.3 目标用户
历史爱好者、策略游戏玩家、教育场景使用者

## 2. 功能需求

### 2.1 游戏核心机制
- **回合制主循环**：每回合玩家处理一个事件
- **事件系统**：随机抽取历史事件，每个事件包含2-3个选项
- **国家资源系统**：人口、黄金、军队、稳定度四个属性
- **连锁记忆效应**：根据历史选择调整后续事件权重或解锁隐藏事件

### 2.2 文明选择系统
- 至少3个可选文明：罗马、汉朝、埃及
- 每个文明有不同的：
  - 起始年份
  - 初始资源数值
  - 事件池内容
  - 历史纪年法

### 2.3 纪年法系统
- 罗马：AUC建城纪年 + 公元年份
- 汉朝：皇帝年号 + 干支 + 公元年份
- 埃及：法老纪年 + 公元年份
- 自动随文明切换

### 2.4 UI展示层
- 当前年份与文明显示
- 四个属性数值条
- 事件卡片（标题+描述+选项按钮）
- 可折叠日志面板（记录最近5条决策）
- 游戏结束面板

### 2.5 游戏结束条件
- 任一属性≤0
- 到达终止年份（公元500年）
- 展示统治总结和评分

## 3. 非功能需求

### 3.1 技术约束
- 纯HTML/CSS/原生JS
- 不引入任何第三方库或框架
- 单文件，可直接双击打开运行

### 3.2 代码组织
- IIFE或模块暴露模式
- 清晰分离四层架构
- 避免全局变量污染
- 逻辑与视图解耦

### 3.3 可量化验证标准
- 事件池≥12个事件
- 可选文明≥3个
- 日志记录最近5条决策
- 覆盖古典、中世纪两个时期

## 4. 交互设计

### 4.1 游戏流程
1. 文明选择界面
2. 游戏主界面（回合循环）
3. 事件处理→属性更新→下一回合
4. 游戏结束→总结展示

### 4.2 用户操作
- 点击选项按钮做出决策
- 展开/折叠日志面板
- 游戏结束后可重新开始
