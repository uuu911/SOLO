## 2. 模块详细设计

### 2.1 audioManager.js

#### 职责
- 负责音频文件的异步加载
- 基于Web Audio API实现播放控制
- 提供精确的当前播放时间查询

#### 核心API
```javascript
class AudioManager {
  async loadAudio(url)      // 加载音频文件
  play()                    // 开始播放
  pause()                   // 暂停播放
  getCurrentTime()          // 获取当前播放时间(秒)
  getDuration()             // 获取音频总时长
  isPlaying()               // 是否正在播放
}
```

### 2.2 beatmapParser.js

#### 职责
- 异步拉取beatmap.json文件
- 解析JSON数据为标准化音符对象
- 控制台输出解析后的音符总数

#### 节拍地图JSON格式
```json
{
  "metadata": {
    "title": "歌曲名称",
    "artist": "艺术家",
    "bpm": 120
  },
  "notes": [
    {
      "time": 1.5,      // 音符时间(秒)
      "lane": 0,        // 轨道索引 0-3 (D/F/J/K)
      "type": "normal"
    }
  ]
}
```

#### 输出音符对象
```javascript
{
  id: number,           // 唯一标识
  time: number,         // 判定时间(秒)
  lane: number,         // 轨道索引 0-3
  type: string,         // 音符类型
  hit: boolean,         // 是否已被判定
  judgment: string      // 判定结果
}
```

### 2.3 gameCore.js

#### 职责
- 游戏主循环(requestAnimationFrame)
- Canvas渲染所有视觉元素
- 键盘输入处理与防抖
- 音符判定逻辑
- 游戏状态管理(分数、生命值、连击)

#### 核心状态
```javascript
{
  score: number,            // 当前分数
  combo: number,            // 当前连击
  maxCombo: number,         // 最大连击
  health: number,           // 生命值(0-100)
  offset: number,           // 节拍偏移(ms)
  isGameOver: boolean,      // 游戏是否结束
  isPaused: boolean,        // 是否暂停
  keys: {                   // 按键状态
    d: boolean,
    f: boolean,
    j: boolean,
    k: boolean
  }
}
```

#### 判定逻辑
```javascript
时间偏差 = |实际按键时间 - 音符时间 + 偏移|
- Perfect: 偏差 ≤ 50ms
- Great:   50ms < 偏差 ≤ 100ms
- Good:    100ms < 偏差 ≤ 150ms
- Miss:    偏差 > 150ms 或 漏击
```

## 3. 渲染系统

### 3.1 Canvas配置
- 推荐尺寸：800x600像素
- 轨道宽度：每个轨道150px
- 判定线位置：Y=500px
- 音符速度：可配置，默认500px/秒

### 3.2 渲染层次
1. 背景层
2. 轨道层
3. 音符层
4. 判定线层
5. 按键反馈层
6. UI层(分数、生命条、连击)
7. 判定特效层

## 4. 输入系统

### 4.1 按键映射
- D键 → 轨道0
- F键 → 轨道1
- J键 → 轨道2
- K键 → 轨道3
- 左/右方向键 → 调整节拍偏移

### 4.2 按键防抖
- 使用对象记录每个音符的判定状态
- 已判定的音符不再参与判定
- 按键按下时只处理最近的未判定音符

## 6. 性能优化

- 使用requestAnimationFrame实现60fps渲染
- 音符对象池复用
- 离屏Canvas缓存静态元素
- 只渲染可视区域内的音符
