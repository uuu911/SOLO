# 音乐节奏游戏产品需求文档 (PRD)

## 1. 项目概述
### 1.1 项目背景
创建一个四轨道下落式音乐节奏游戏，完全基于HTML5 Canvas和Web Audio API实现，无需任何构建工具，可直接在浏览器中运行。

### 1.2 目标用户
- 音乐节奏游戏爱好者
- 浏览器游戏开发者
- Web技术学习者

### 1.3 核心价值
- 纯前端实现，无需后端支持
- 模块化架构，易于扩展和维护
- 精确的节拍判定系统
- 流畅的游戏体验

## 2. 功能需求

### 2.1 核心玩法
- **四轨道下落式音符**：音符从屏幕上方沿四个轨道垂直下落
- **按键映射**：D/F/J/K对应四个从左到右的轨道
- **判定系统**：
  - Perfect：时间偏差≤50ms
  - Great：偏差50~100ms
  - Good：偏差100~150ms
  - Miss：偏差>150ms或漏击

### 2.2 游戏状态管理
- **生命值系统**：初始100%
  - Perfect/Great：不扣血
  - Good：恢复3%
  - Miss：扣除15%
- **连击系统**：连续正确击打递增，Miss时清零
- **计分系统**：
  - Perfect：100分
  - Great：60分
  - Good：30分
  - 连击加成：每10连击额外+5分
- **评级系统**：
  - S级：≥95%总分
  - A级：≥80%总分
  - B级：≥65%总分
  - C级：<65%总分

### 2.3 音频系统
- 基于Web Audio API实现
- 支持./music.mp3音频文件
- 精确的音频时间查询

### 2.4 节拍地图
- 从./beatmap.json加载节拍数据
- 支持四个轨道的音符
- 支持同一时间点的多个音符（和弦）

### 2.5 特殊功能
- **节拍偏移调整**：范围±200ms，步长5ms，方向键控制
- **按键动画**：按下时轨道高亮，松开恢复
- **按键防抖**：每个音符只能判定一次
- **游戏结束条件**：
  - 生命值归零时立即冻结
  - 歌曲结束且所有音符飞出屏幕后显示结算

## 3. UI/UX设计

### 3.1 界面元素
- Canvas绘制所有视觉元素
- 轨道、判定线、音符
- 连击数、生命条、评分特效
- 节拍偏移实时显示

### 3.2 视觉风格
- **霓虹科技风**：深色背景配合霓虹色彩
- **轨道颜色**：D(红)/F(蓝)/J(绿)/K(紫)
- **判定特效**：不同判定等级不同颜色
- **流畅动画**：音符下落、按键反馈、判定特效

## 4. 技术架构

### 4.1 模块划分
- **audioManager.js**：音频加载、播放、时间查询
- **beatmapParser.js**：节拍JSON解析
- **gameCore.js**：主循环、渲染、输入、判定、状态管理
- **index.html**：入口文件

### 4.2 技术栈
- HTML5 Canvas
- Web Audio API
- ES Modules
- 纯原生JavaScript（无框架）

## 5. 验证标准
- Perfect判定窗口精确到50ms
- 生命值归零游戏冻结
- 偏移调整步长5ms
- 解析后控制台输出音符总数
- Miss后连击立即归零
- 结算后控制台显示分数明细
