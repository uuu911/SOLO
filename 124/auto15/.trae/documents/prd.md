# MOBA核心玩法演示 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
创建一个基于HTML5 Canvas的MOBA（多人在线战术竞技）核心玩法演示，展示英雄、技能、小兵、防御塔之间的线上交互逻辑。

### 1.2 项目目标
- 实现完整的MOBA单线对抗玩法
- 展示清晰的模块划分和接口设计
- 提供可直接运行的单文件HTML演示
- 60FPS流畅运行体验

### 1.3 目标用户
游戏开发者、前端开发者、MOBA游戏爱好者

---

## 2. 功能需求

### 2.1 核心游戏机制

#### 2.1.1 英雄系统
- **接口定义**: IChampion接口包含移动、普攻、释放技能、更新状态等方法
- **实现要求**: 至少2个不同英雄，每个拥有1个普攻和1个特殊技能
- **英雄差异**:
  - 英雄A: 线性非指向性弹道技能（如Q版射击）
  - 英雄B: 圆形范围地面施放技能（如AOE爆炸）

#### 2.1.2 技能系统
- **接口定义**: ISkill接口包含执行技能、获取冷却时间、获取技能范围等方法
- **技能机制**:
  - 普攻: 攻击前摇，面朝目标，125-175单位射程
  - 特殊技能: 施法前摇，技能范围指示器，10-15秒冷却
  - 技能不可取消

#### 2.1.3 小兵系统
- **生成机制**: 双方基地每30秒生成一波小兵
- **移动逻辑**: 沿兵线朝敌方基地移动
- **攻击优先级**:
  1. 距离最近的敌方小兵
  2. 攻击范围内的防御塔/英雄
  3. 不可无视近身小兵直接追英雄

#### 2.1.4 防御塔系统
- **属性**: 300单位射程（大于英雄普攻射程）
- **攻击逻辑**:
  - 默认: 攻击最先进入射程的敌方小兵
  - 仇恨机制: 敌方英雄在塔范围内攻击我方英雄时，立即切换目标攻击该英雄
  - 恢复逻辑: 英雄脱离射程或死亡后恢复默认

#### 2.1.5 地图系统
- **视角**: 俯视视角
- **结构**: 单条兵线，双方基地位于两端
- **建筑**: 每方1座防御塔 + 1个可攻击水晶
- **地形**: 道路之间为不可通行区域

### 2.2 玩家操作系统

#### 2.2.1 输入控制
- **移动**: 鼠标右键点击地面
- **普攻**: 按A键后左键点击目标（显示攻击范围圈）
- **技能**: 按Q键后鼠标指向显示范围指示器，左键释放
- **输入状态机**: 防止操作冲突（如移动中按Q应准备技能而非直接释放）

#### 2.2.2 UI显示
- **生命条**: 所有单位顶部显示
- **技能UI**: 英雄技能小图标 + 冷却倒计时（精确到小数点后1位）
- **技能指示器**: 随鼠标实时更新形状和有效区域

### 2.3 AI系统
- **补刀逻辑**: 对低血量小兵优先普攻
- **撤退逻辑**: 血量低于30%时向己方塔下撤退
- **技能释放**: 有合适目标时尝试释放技能

## 3. 技术规格

### 3.1 性能指标
- **帧率**: 稳定60FPS
- **驱动方式**: requestAnimationFrame
- **英雄速度**: 200-350单位/秒

### 3.2 数值设定
| 参数 | 数值范围 |
|------|----------|
| 英雄移动速度 | 200-350单位/秒 |
| 普攻射程 | 125-175单位 |
| 技能冷却 | 10-15秒 |
| 小兵生成间隔 | 30秒 |
| 防御塔射程 | 300单位 |

### 3.3 代码结构
- **模块化**: ES模块或立即执行函数划分职责
- **单文件**: 整合为单个HTML文件，可直接双击打开
- **接口与实现分离**: 接口定义和具体实现分属不同模块
